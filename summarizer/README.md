# Summary of architecture

This is meant to be a microservice that tracks the number of people that enter or leave an alert. If the alert is new then the summary should say all people have come in. Otherwise calculate the difference for who came in and who left

# Dictionary
Alert - A bad thing in the world that we want to tell our customers about eg: hurricane, fire etc.
Contact - A person potentially affected by an alert
Summary - A small chunk of text summarizing some data about an alert
